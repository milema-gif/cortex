  <action>
1. **Create src/tools/reconcile.ts:**
   - Import McpServer, Database, z (from zod)
   - Import reconcileRetry, reconcileDrop, reconcileAck from ../core/reconcile.js
   - Import computeHealth from ../core/health.js
   - Import log from ../lib/logger.js
   - Export `registerReconcileTool(server: McpServer, db: Database.Database): void`
   - Register tool `cortex_reconcile` with description: "Reconcile sync failures. Actions: retry(observation_id), drop(observation_id), ack (acknowledge blocked state for this session)."
   - Input schema using zod:
     ```
     { action: z.enum(["retry", "drop", "ack"]),
       observation_id: z.number().optional() }
     ```
   - Handler logic:
     - If action is "retry" or "drop" and observation_id is missing: return error "observation_id required for {action}"
     - If action is "retry": call reconcileRetry(db, observation_id)
     - If action is "drop": call reconcileDrop(db, observation_id)
     - If action is "ack": call reconcileAck(db)
     - Return MCP content with JSON stringified ReconcileResult
     - On error: return isError: true with error message

2. **Modify src/core/sync.ts -- add execution gate:**
   - Import computeHealth from ./health.js
   - Import isAcked from ./reconcile.js
   - At the TOP of `syncNewObservations`, before any work:
     ```typescript
     const health = computeHealth(db);
     if (health === 'blocked' && !isAcked()) {
       log("warn", "sync: BLOCKED -- mutating operation skipped. Run cortex_reconcile to resolve.");
       return;
     }
     ```
   - At the TOP of `embeddingBackfill`, same gate:
     ```typescript
     const health = computeHealth(db);
     if (health === 'blocked' && !isAcked()) {
       log("warn", "backfill: BLOCKED -- mutating operation skipped. Run cortex_reconcile to resolve.");
       return 0;
     }
     ```

3. **Modify src/server.ts:**
   - Import registerReconcileTool from ./tools/reconcile.js
   - Import computeHealth from ./core/health.js
   - Add `registerReconcileTool(server, db);` after the other registerXxxTool calls
   - After `runMigrations(db)` and before the mode switch, add startup health check:
     ```typescript
     const startupHealth = computeHealth(db);
     if (startupHealth === 'blocked') {
       log("warn", "=== CORTEX HEALTH: BLOCKED ===");
       log("warn", "Parked sync failures require reconciliation.");
       log("warn", "Run cortex_reconcile with retry, drop, or ack to proceed.");
       log("warn", "Mutating operations (sync, backfill) will be skipped until resolved.");
       log("warn", "==============================");
     } else if (startupHealth === 'degraded') {
       log("info", `Cortex health: degraded (sync failures exist). Use cortex_status for details.`);
     }
     ```
